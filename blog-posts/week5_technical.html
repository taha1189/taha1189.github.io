Classes in Object Oriented Programming

Before Object Oriented Programming there was Procedural Programming. The fundamental underlying concept of Object Oriented Programing that differentiates it from the older style of programming is the use objects which allow for re-usability of code.
Object Oriented Programming (OOP), utilizes data structures known as "objects" which incorporate procedures known as "methods".

TO fully understand the fundamentals of OOP there are 3 basics that need to be covered.
  • Objects
    • Everything in OOP can be represented as an object. Each object is characterized by 2 traits, that state it is currently in, and actions available to the object. Objects in programming are often used to represent real-world objects.
  • Classes
    • Classes can be described as the blueprint from which objects are created. The classs represents the object as a whole.
  • Inheritance
    • Inheritance offers a concise way to organize and structure your software. Although Classes offer the blueprint for your objects, objects can still differ. For example, the objects of class Bike will have similar qualities and functions however, Inheritance can allow for differences in the Bike suhc as mountain bikes or tandem bikes.
